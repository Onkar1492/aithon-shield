# Feature delivery output rules (MVP1 pre-launch)

These rules apply to every feature shipped in the pre-launch plan. The agent runs all builds, tests, and servers; the product owner does not run terminal commands.

## Required for each feature

1. **Plain-language summary** — Explain what was implemented in simple terms (no jargon wall).
2. **Category 1 (Cat 1) — UI** — What the user can see or click in the app.
3. **Category 2 (Cat 2) — Backend** — APIs, server logic, database, or config the user does not see directly.
4. **Testing performed** — What was run or checked (including automated checks and browser smoke tests when applicable).
5. **Step-by-step validation** — Ordered steps for the product owner to confirm behavior in the running app (no CLI required).

## Optional

- Screenshots when UI changes are material.
- Links to related docs under `MVP2-Final/` when relevant.

## Naming

- Pre-launch implementation work is tracked under the **Pre-Launch Implementation Plan** (consumer MVP1: Free / Starter / Pro; enterprise-only code archived to MVP2-Final).
