## What Was Implemented

### Schema Changes

Added 4 new fields to all three scan tables (`mvpCodeScans`, `webAppScans`, `mobileAppScans`):

1. **`scanProgress`** (integer, nullable)
   - Range: 0-100
   - Represents scan completion percentage
   - Default: `null` (not started)

2. **`scanStage`** (text, nullable)
   - Current stage of the scan
   - Examples: 
     - MVP: "Cloning repository", "Running SAST analysis", "Scanning dependencies"
     - Web: "Crawling web application", "Testing OWASP Top 10", "Analyzing SSL/TLS"
     - Mobile: "Downloading app", "Extracting archive", "Analyzing binary"

3. **`scanError`** (text, nullable)
   - Error message if scan fails
   - Default: `null`

4. **`cancellationRequested`** (boolean)
   - Flag to request scan cancellation
   - Default: `false`
